ziploader.js

ziploader.js is a library for loading ZIP format files and applying them to the DOM.

Instead of CSS sprites

CSS sprites are one of the most well-known ways to optimize a website, reducing the number of HTTP requests by combining multiple images into a single sprite map.
While they can be combined into a single image, they are implemented as the background of an element, not as an img element, which is a traditional image.

In ziploader.js, after loading the zip file, the src attribute of each img element is rewritten and displayed, so it can be handled as an img element.
Also, since each image is a separate image, it is possible to enlarge or reduce them individually.

This can be done simply by writing HTML and JavaScript as shown below:

HTML

First, write the URL of the zip file in the data-ziploader-archive attribute.
Then write the name of the file within the zip file in the data-ziploader-filename attribute.

<img data-ziploader-archive="images/svg32.zip" data-ziploader-filename="access_bus.svg" width="4" height="4">
<img data-ziploader-archive="images/svg32.zip" data-ziploader-filename="access_train.svg" width="8" height="8">
<img data-ziploader-archive="images/svg32.zip" data-ziploader-filename="access_walk.svg" width="16" height="16">
<img data-ziploader-archive="images/svg32.zip" data-ziploader-filename="arrow_c_down.svg" width="32" height="32">
<img data-ziploader-archive="images/svg32.zip" data-ziploader-filename="arrow_c_left.svg" width="64" height="64">
<img data-ziploader-archive="images/svg32.zip" data-ziploader-filename="arrow_c_right.svg" width="128" height="128">
<img data-ziploader-archive="images/svg32.zip" data-ziploader-filename="arrow_c_up.svg" width="256" height="256">

JavaScript: Process all img elements with the data-archive attribute with ziploader.js

document.addEventListener('DOMContentLoaded', function() {
    var zl = new ZipLoader();
    zl.applyElementAll('img[data-ziploader-archive]', 'src');
}, false);

During development, refer to the files directly instead of using zip files.

It would be a pain to recreate the zip file every time you develop.
So, by using the js file built for development instead, you can directly reference the files from the directory with the same name as the zip file.
For example, in the above example,

  • images/svg32.zip/access_bus.svg

If you place the file like this, the image will be displayed as is.

Fallback to PNG for environments that don't support SVG

Since SVG cannot be displayed on Android 2.x, you may want to display a PNG file instead.
Even in such a case, by calling ZipLoader#setSVGFallback(true); and then applyElementAll, the file with the .svg extension will be automatically replaced with .png on devices that do not support it.
